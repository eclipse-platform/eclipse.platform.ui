Eclipse 3.3 Debug Platform Build Notes 

Summary of API changes in 3.3

  • added DebugPlugin.ATTR_CONSOLE_ENCODING - this launch attribute specifies the encoding used by the console. When unspecified the default system encoding is used. This constant replaces the now deprecated IDebugUIConstants.ATTR_CONSOLE_ENCODING.
  • added org.eclipse.debug.ui.actions.IWatchExpressionFactoryAdapterExtension - this is an interface that can be optionally implemented by IWatchExpressionFactoryAdapter's. Allows the Create Watch Expression action to be dynamically enabled based on the selected variables.
  • deprecated IDebugUIConstants.ATTR_CONSOLE_ENCODING.
  • added DebugUITools.getLastLaunch(String id) - provides access to the last launch in each launch group.

Summary of Extensible Launch Option conversion/changes in 3.3

  • Added method ILaunchConfigurationType.getContributorName() -- returns the name of the contributing plug-in
  • Added method ILaunchConfigurationType.getImageDescriptorPath() -- returns path of the image or null
  • Added attrbute 'icon' in launchConfigurationTypes schema, in favour of contributing a seperate extension launchConfigurationTypeImages
  • Added tab id filed to AbstractLaunchConfigurationTab class
  • Provided tab ids for all debug provided tabs
  • Added method openLaunchConfigurationEditDialog -- opens the launch configuration dialog on a configuration to be edited and launching can be continued
  • Added getAttribute() to ILaunchConfiguration, which returns a java.util.Set
  • Added a new subclass of LaunchConfigurationsDialog, LaunchConfigurationEditDialog which allows a configuration to be edited on the fly with a continue button.
  • Added method getSetAttribute to LaunchConfigurationInfo
  • Added method getTabExtensions which returns the list of launchConfigurationTab extension proxies.
  • Added method getRelativeTabId to LaunchConfigurationTabExtension -- returns the id of the tab that this tab should lbe placed relative to i.e.immediately after.
  • Added a select options Link to LaunchConfiguraitonTabGroupViewer, which is used to resolve problems
    • with duplicate and unavailable launch delegates
  • Added new status handler LaunchDelegateNotAvailabaleHandler, which prompts a user to select a differnet launch delegate to resolve problems with the current one.
  • Added method getLaunchDelegates to LaunchManager -- returns all of the contributed launch delegates.
  • Added method getLaunchDelegates(String typeid) to LaunchManager -- returns the launch delegates contributed for the specific launch configuration type id.
  • Added method initializeLaunchDelegates to LaunchManager -- initially creates the listing of contributed launch delegates
  • Added dialog class SelectLaunchModesDialog -- used to allow users to select a differing launch delegate in the event there is a problem with the current one.
  • Added LAUNCH_MODES constant to LaunchConfiguration
  • Added class AbstractLaunchModeConfigurationTab to provide some default implememtation for contributing a launch configuration tab that provides a mode attribute.
  • Added class LaunchConfigurationTabGroupWrapper which provides a wrapper for a contributed launch group and contributed tabs for that launch group.
  • Added constant EXTENSION_POINT_LAUNCH_TABS to IDebugUIConstants for finding contributed launch configuration tabs.
  • Added interface ILaunchDelegate which provides API for access to a launch delegates extension point.
  • Added attribute 'name' to the launchDelegates schema, providing a human readable name for the delegate. To be used in duplicate resolution
  • Added method getModes to ILaunchConfiguration -- returns the set of modes that have been set on the launch configuration.
  • Added method getDelegates(Set) to ILaunchConfigurationType to return an array of ILaunchDelegate instances for the associated type.
