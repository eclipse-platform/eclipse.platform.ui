Cheat Sheet Content File XML Format

This document describes the cheat sheet content file structure as a series of DTD fragments.

cheatsheet Element

<?xml encoding="US-ASCII"?> 
<!ELEMENT cheatsheet (intro, item+)> 
<!ATTLIST cheatsheet 
  title               CDATA #REQUIRED
>

The <cheatsheet> element defines the body of the cheat sheet content file. <cheatsheet> attributes are as follows:

  • title - the title of the cheat sheet 
<?xml encoding="US-ASCII"?> 
<!ELEMENT intro ()> 
<!ATTLIST intro 
  href                CDATA #IMPLIED 
>

intro Element

The <intro> element is used to describe the cheat sheet introduction to be displayed. The text between the <intro> and </intro> tags is the introduction. <intro> attributes are as follows:

  • href - the cheat sheet's optional concept topic; when the user clicks the introduction's help link, this document is shown in the help browser

item Element

<?xml encoding="US-ASCII"?> 
<!ELEMENT item (subitem*)> 
<!ATTLIST item 
  title               CDATA #REQUIRED
  kind                ("action" | "manual" | "both")
  skip                ("true" | "false") "false"
  class               CDATA #IMPLIED
  pluginId            CDATA #IMPLIED
  actionparam0        CDATA #IMPLIED
  actionparam1        CDATA #IMPLIED
  ...
  actionparam9        CDATA #IMPLIED
  href                CDATA #IMPLIED
>

Each <item> element describes one top-level item (or step) in a cheat sheet. The <item> is either simple, in which case the text between the <item> and </item> tags is displayed in the UI as the item's description, or is complex, in which case the <subitem> elements between the the <item> and </item> tags break the step down into sub-steps. <item> attributes are as follows:

  • title - title of the cheat sheet item
  • kind - the kind of step
    • "action" - the step has an accompanying computed-provided action ("class" and "pluginId" attributes); the UI generally shows a button that the user can press to invoke that action
    • "manual" - the step involves the user doing something on their own; the UI generally shows a button that the user can press to indicate that they have done whatever they were supposed to do
    • "both" - the step has both a computer-provided action but also requires the user to manually perform some steps and confirm that they have done so
  • skip - skip="true" means that the step can be skipped; the UI generally shows a button that the user can press to indicate that they are skipping this step
  • class - the fully-qualified name of the Java class implementing org.eclipse.ui.cheatsheets.ICheatSheetAction or its superinterface org.eclipse.jface.action.IAction; the pluginId attribute must be present whenever the class attribute is present
  • pluginId - the id of the plug-in which contains the Java class of the action class; this attribute must be present whenever the "class" attribute is present
  • actionparamN - for action classes that implement org.eclipse.ui.cheatsheets.ICheatSheetAction, the string values of these attributes are passed to the action when it is invoked. You can pass up to 10 parameters to a cheat sheet item's action (actionParam0, actionParam1, etc.) [TODO - need to complete spec for variables determined at cheat sheet execution time; e.g., actionParam0="${var1}"]
  • href - the cheat sheet items's optional task documentation link; when the user clicks the item's help link, this document is shown in the help browser

subitem Element

<?xml encoding="US-ASCII"?> 
<!ELEMENT subitem EMPTY> 
<!ATTLIST subitem 
  label               CDATA #REQUIRED
  kind                ("action" | "manual" | "both")
  skip                ("true" | "false") "false"
  class               CDATA #IMPLIED
  pluginId            CDATA #IMPLIED
  actionparam0        CDATA #IMPLIED
  actionparam1        CDATA #IMPLIED
  ...
  actionparam9        CDATA #IMPLIED
>

Each <subitem> element describes a sub-item (or sub-step) in a cheat sheet. A <subitem> carries a simple text label, but has neither a lengthy description nor further sub-items. There should be a least 2 sub-items within the item. Each sub-item has its own action buttons.

Note that help and extension items buttons are generally presented only on items, not sub-items.

Unlike items, which must be followed in strict sequence, the sub-items of a given item can be performed in any order. All sub-items within an item have to be attempted (that is, performed, skipped, or completed) before progressing to the next item. (Which means actions that must be performed in a required sequence cannot be represented as sub-items.) 

<subitem> attributes are as follows:

  • label - title of the cheat sheet sub-item
  • kind - the kind of step (
    • "action" - the step has an accompanying computed-provided action ("class" and "pluginId" attributes); the UI generally shows a button that the user can press to invoke that action
    • "manual" - the step involves the user doing something on their own; the UI generally shows a button that the user can press to indicate that they have done whatever they were supposed to do
    • "both" - the step has both a computer-provided action but also requires the user to manually perform some steps and confirm that they have done so
  • class - the fully-qualified name of the Java class implementing org.eclipse.ui.cheatsheets.ICheatSheetAction or its superinterface org.eclipse.jface.action.IAction; the pluginId attribute must be present whenever the class attribute is present
  • actionparamN - for action classes that implement org.eclipse.ui.cheatsheets.ICheatSheetAction, the string values of these attributes are passed to the action when it is invoked. Basic Elements and Attributes

The following snippet shows an example of a cheat sheet XML that is provided in the cheat sheet sample plugin. 

<?xml version="1.0" encoding="UTF-8" ?>  
  
<cheatSheet title="Hello World Demo" version="1.0"> 
  
<intro  
      href="/org.eclipse.ui.cheatsheets.doc/tasks/tcheatst.htm"> 
Welcome to the hello world cheat sheet. 
This cheat sheet will help you to build the famous "hello world"
application and try it out. You will create a java project, and
a Java class that will print "hello world" in the console when run. 
Let's get started! 
</intro> 
  
<item 
      id="0" 
      title="Open the Java Perspective"  
      href="/org.eclipse.platform.doc.user/concepts/concepts-4.htm" 
      kind="manual"  
      pluginId="org.eclipse.ui.cheatsheet.sample" 
      class="org.eclipse.ui.cheatsheets.sample.pattern.actions.LaunchPatternSelectionDialogAction">
To begin, open the Java Perspective in the Eclipse workbench.  
You may do this by clicking window->open perspective->Java in the
toolbar at the top of the workbench. When you have done this, press
the "click when done" button below.
</item> 
  
</cheatSheet>

Item extensions

If you need extend an item’s assistance capabilities to launch additional support tools besides help topics, you can use the item extension mechanism to display additional custom controls to the left of help link  button. To accomplish this, you need to do these two things:Use the extension point to extend the cheat sheet <item> In the cheat sheet content XML file, use the newly defined <item> attribute that supports the item extension.

Extension point for item extensions

In the cheat sheet content plugin's plugin.xml file, use the extension point, org.eclipse.ui.cheatsheets.cheatSheetItemExtension, as follows: <extension point="org.eclipse.ui.cheatsheets.cheatSheetItemExtension">         <itemExtension                 itemAttribute="extensionlink"                 class="org.example.actions.ItemExtensionAction">         </itemExtension> </extension> In the above example extension point, the itemAttribute="" attribute specifies the name of the attribute to use when defining the <item> that will have the item extension included. You can use the same item extension for more than one <item>. The class="" attribute is a class that implements the org.eclipse.ui.cheatsheets.ICheatSheetItemExtensionElement interface. This class is called to handle the following: · Parsing of the information in the cheat sheet content XML file. · Creation of the custom control.

Defining <item> that uses the item extension

In the cheat sheet content XML file, the <item> element that you wish to have the item extension needs to use the item extension attribute. The value of the itemAttribute in the <itemExtension> element is the attribute that you will use in the <item> definition. Here is an example of the <item> element that has the extensionlink attribute as defined in the above extension point example. <item         id="0"         href="/org.eclipse.platform.doc.user/concepts/concepts-4.htm"         title="Open the Java Perspective"         extensionlink=”input data required by extension”        actionphrase="2"        pluginId="org.eclipse.ui.cheatsheet.sample"         class="org.eclipse.ui.cheatsheets.sample.pattern.actions.LaunchPatternSelectionDialogAction">To begin, open the Java Perspective in the Eclipse workbench. You may do this by clicking window->open perspective->Java in the toolbar at the top of the workbench.  When you have done this, press the "click when done" button below.</item>

Additional notes

Text formatting and line breaks In the cheat sheet content XML file keep all the text in the same line as the <intro> or <item> tag. At execution time, the cheat sheet will automatically wrap the text to make it show up correctly in the cheat sheet. If you break the text into another line in the cheat sheet content XML file, you will force a line break in the displayed cheat sheet.
