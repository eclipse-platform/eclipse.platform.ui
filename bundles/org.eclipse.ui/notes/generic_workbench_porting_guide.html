Eclipse 3.0 Porting Guide
Generic Workbench

Last revised July 22, 2003

[Under construction]

Note that the changes described here are still at the proposal stage.

In 3.0, the Eclipse Platform UI changed to separate out generic workbench from IDE specifics. This resulted in changes to API packages and extension points. This document describes how to port existing Eclipse 2.1 plug-ins to 3.0 in light of these changes.

  • Plug-in changes
  • Extension points that have moved to different plug-ins
  • API packages that have moved to different plug-ins
  • API packages that were subdivided and are now split across plug-ins
  • API classes and interface that have changed

Plug-in changes

The org.eclipse.ui plug-in which used to be the main Platform UI plug-in, now contains just the API and extension points for the generic workbench. Optional and IDE-specific API and extension points have moved to other plug-ins; IDE-specific functionality is now in the org.eclipse.ui.ide plug-in; text editors are now in the org.eclipse.ui.workbench.texteditor and org.eclipse.ui.editors plug-ins; the properties, content outline, and navigator views are now in their own separate plug-ins: org.eclipse.ui.views.properties, org.eclipse.ui.views.contentoutline, and org.eclipse.ui.views.navigator respectively.

The org.eclipse.ui.workbench plug-in was deleted. This was an internal UI plug-in, which client plug-ins would not ordinarily reference (it has no extension points, and all its API was re-exported by the org.eclipse.ui plug-in). The classes in this plug-in's API packages are now in one of the other UI plug-ins. The plug-in manifest file for a client plug-in that explicitly requires the org.eclipse.ui.workbench plug-in should be changed to require (a sufficient subset of) these plug-ins instead: org.eclipse.ui, org.eclipse.ui.editors, org.eclipse.ui.views.navigator, org.eclipse.ui.views.properties, org.eclipse.ui.views.contentoutline, org.eclipse.ui.ide.

The org.eclipse.ui.views plug-in was deleted. This was an internal UI plug-in, which client plug-ins would not ordinarily reference (it has no extension points, and all its API was re-exported by the org.eclipse.ui plug-in). The classes in this plug-in's API packages are now in one of the other UI plug-ins. The plug-in manifest file for a client plug-in that explicitly requires the org.eclipse.ui.views plug-in should be changed to require (a sufficient subset of) these plug-ins instead: org.eclipse.ui.views.navigator, org.eclipse.ui.views.properties, org.eclipse.ui.views.contentoutline, org.eclipse.ui.ide.

Extension points that have moved to different plug-ins

The following extension points have moved to different plug-ins, causing their extension point ids to change.

The following extension points moved from the org.eclipse.ui plug-in to the org.eclipse.ui.ide.plug-in. In the extension element in the plug-in manifest file, change references to the extension points "org.eclipse.ui.foo" to refer to "org.eclipse.ui.ide.foo" instead.

  • org.eclipse.ui.markerImageProvider
  • org.eclipse.ui.capabilities
  • org.eclipse.ui.exportWizards
  • org.eclipse.ui.importWizards
  • org.eclipse.ui.markerHelp
  • org.eclipse.ui.markerImageProviders
  • org.eclipse.ui.markerResolution
  • org.eclipse.ui.newWizards
  • org.eclipse.ui.projectNatureImages
  • org.eclipse.ui.resourceFilters
  • org.eclipse.ui.workingSets

The following extension points moved from the org.eclipse.ui plug-in to the org.eclipse.ui.workbench.texteditor plug-in. In the extension element in the plug-in manifest file, change references to the extension points "org.eclipse.ui.foo" to refer to "org.eclipse.ui.workbench.texteditor.foo" instead.

  • org.eclipse.ui.markerUpdaters
  • org.eclipse.ui.documentProviders

The following extension points moved from the org.eclipse.help plug-in to the org.eclipse.help.base plug-in. In the extension element in the plug-in manifest file, change references to the extension points "org.eclipse.help.foo" to refer to "org.eclipse.help.base.foo" instead.

  • org.eclipse.help.browser
  • org.eclipse.help.luceneAnalyzer
  • org.eclipse.help.webapp
