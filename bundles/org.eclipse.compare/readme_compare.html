Build 032

What's new in this drop

API changes

  • deprecated ViewerPane; use CompareViewerSwitchingPane instead
  • deprecated ByteContentAccessor and IByteContentAccessor. Use IStreamContentAccessor instead.
  • deprecated IDiffParent; Use IDiffElement instead.
  • Moved more classes to internal package.
  • removed byte[] argument from IEditable.replace. Replace only handles add, remove, and copy contents. For setting a contents use IEditable.setContents.
  • Renamed IByteContentChangedListener to IContentChangedListener.
  • removed methods addChangeListener and removeChangeListener from IByteContentAccessor. Use IContentChangedProvider instead.
  • Deprecated IDiffConstants.INCOMING, IDiffConstants.OUTGOING. Use LEFT and RIGHT instead. The interpretation of LEFT and RIGHT (e.g. Incoming and Outgoing) is left to the client.
  • Changed return type of  DiffContainer.findChild(String name) from DiffContainer to IDiffElement.
  • Changed return type of IDiffContainer.getChildren from Iterator to Object[].

    •  

API Additions

  •  added method ResourceNode.createChild(IResource child)

    • (can be used to filter children)
  • added method setContents(byte[] ...) to IEditable
  • added interface IContentChangedProvider.

    •  

Other highlights

  • Panes within compare editor can be resized
  • Replace from history buffer.
  • Support for method level editions.

Known deficiencies

  • lots

Build 021

What's new in this drop

API changes

  • Harmonized usage of the compare directions: Mine/Yours/Your, Incoming/Outgoing, Left/Right. Everywhere Left/Right is used.

    • API changes: ContentMergeViewer, ImageMergeViewer, TextMergeViewer, IMergeViewerContentProvider, MergeViewerContentProvider, ICompareConfiguration, CompareConfiguration, IThreeWayInput, StructureDiffViewer, DiffNode, DiffTreeViewer
  • changed signature of IStructureCreator.save from void save(Object input, IDocument document) to void save(Object input, IStructureComparator structure)
  • made com.ibm.eclipse.ui.compare.structuremergeviewer.ArrayIterator package private.

API Additions

  • Added new classes CompareOp, ResourceCompareOp, HistoryCompareOp

    • All comparison operations can now be written as subclasses of CompareOp. CompareUIPlugin.runCompareOp() generically opens a CompareEditor for it. ResourceCompareOp implements a universal two/threeway compare on desktop resources. HistoryCompareOp compares a resource with its most recent edition from the history buffer. Another example would be a RepositoryCompareOp.

Other highlights

  • background coloring in the TextMergeViewer no longer depend on StyledText background coloring support.
  • generic copy-left-to-right and copy-right-to-left action now work in second and third structure panes.

    • In the resource pane they just print a message to the console but don't do anything.
  • Progress is shown while a compare runs.
  • Support for comparing jpegs.
  • if resources with unknown types are compared the Compare Plugin tries to guess whether they contain text and whether the TextMergeViewer can be used.

Known deficiencies

  • correct focus/activation handling is still broken.

Problem reports fixed

Problem reports closed
